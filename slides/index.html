The Challenge

Enterprise Requirements

  • Tenant data isolation (RBAC, network, IAM)
  • Zero-downtime deployments
  • Self-service tenant onboarding
  • Cost visibility per tenant
  • Compliance-ready (SOC2 path)

Team Constraints

  • 7-person team, no dedicated platform team yet
  • 10–20 enterprise customers in 12 months
  • 4-week delivery window
  • Must not block product velocity
The decision: Namespace-per-tenant on shared EKS in Phase 1 — operational simplicity wins at this team size. Account-per-tenant is the Phase 2 roadmap when compliance demands it.

Terraform: tenant-isolation

Cloud-layer boundary per tenant. K8s resources (namespace, RBAC, NetworkPolicy) live in Helm + Flux — not here.

module "tenant_acme" {
  source = "./modules/tenant-isolation"

  tenant_name            = "acme"
  environment            = "prod"
  vpc_id                 = module.vpc.vpc_id
  cluster_oidc_issuer_url = module.eks.cluster_oidc_issuer_url
  allowed_namespaces     = ["acme"]
  availability_zones     = ["us-east-1a", "us-east-1b", "us-east-1c"]
  # outputs: iam_role_arn, security_group_id, subnet_ids
  # → consumed by Flux HelmRelease values for the tenant-base chart
}
Provisions: 3 private subnets (per AZ) · Security group (intra-tenant + Istio ingress) · IRSA role scoped to s3://bucket/acme/* + Secrets Manager paths

How I've Done This Before

BP Bifrost — Multi-Cloud AKS + GKE Platform

Situation
BP data scientists each managed individual Windows VMs — unmanaged, outside corporate identity, always-on Azure compute, one overnight training run per day. No cost controls, no visibility, no way to queue work.
Action
Built multi-cloud K8s platform from scratch as sole DevOps engineer: AKS (Azure) for JupyterHub + Entra ID SSO, GKE (Google Cloud) for NVIDIA GPU spot compute with a custom K8s controller watching job queue depth. Launched AKS+GKE together as one logical multi-tenant environment — beta with the data science team, then full org rollout. Namespace-per-team isolation across both clusters, Flux + GitLab GitOps, Istio mesh.
Result
40% Azure cost reduction (eliminated always-on VMs) · Multiple training iterations per day vs. one overnight · 99.9% uptime across multi-cloud in production · All users under Entra ID for the first time.
